# Claude Code 工作規則

## 規則 1: 通訊語言
- **語言**: 繁體中文
- **適用範圍**: 所有與使用者的對話、文件註解、代碼註解
- **說明**: 無論任何情況，都使用繁體中文與使用者溝通

## 規則 2: 檔案管理
- **位置**: 所有產生的檔案必須放在 `docs/` 目錄下
- **適用範圍**: 報告、文件、配置、日誌等所有新產生的檔案
- **說明**: 保持專案根目錄的整潔，所有產出物集中管理

## 規則 3: Git 操作權限
- **責任分工**:
  - Claude Code: 準備代碼修改、提供具體的 git 指令
  - 使用者: 執行所有 git 操作（add、commit、push）
- **流程**:
  1. Claude Code 準備代碼修改和提交訊息
  2. 展示待提交的修改內容給使用者審閱
  3. 提供具體的 git 指令（包含提交訊息）
  4. 用戶確認無誤後執行指令
  5. 用戶決定何時 push 及推送時機
- **具體指令範例**:
  ```bash
  # 查看修改
  git diff <檔案路徑>

  # 添加到暫存區
  git add <檔案路徑>

  # 提交修改
  git commit -m "提交訊息"

  # 推送到遠端
  git push
  ```
- **說明**: 確保用戶對所有 git 操作有完全的控制和審批權
